# C++ Review Course

This repository is a beginner-friendly C++ review course built as a collection of short lesson files. Each file focuses on one idea and includes simple commentary, study notes, and beginner tips so students can learn by reading and experimenting.

## What This Course Covers

This course is designed to help students build confidence with core C++ concepts and to support the following competencies:

### Competency 1: Program Development Process
Students are encouraged to:
- think through a problem before coding
- break problems into smaller parts
- add meaningful comments to code
- test and debug simple programs

### Competency 2: Basic Data Types and Operators
The lessons introduce:
- integers, doubles, characters, booleans, and strings
- variables and meaningful names
- type casting
- arithmetic and basic operators

### Competency 3: Conditional Statements
The course includes lessons on:
- if, else, and else if statements
- logical operators
- comparison operators
- switch, case, and break
- nested conditionals

### Competency 4: Loops
Students will work with:
- while loops
- do-while loops
- for loops
- nested loops

### Competency 5: Functions
The course introduces:
- functions and return values
- function scope
- library functions
- basic function concepts that prepare students for more advanced work

### Competency 6: Structured Data Types
Lessons cover:
- arrays and multi-dimensional arrays
- searching and sorting ideas
- structures
- classes, encapsulation, inheritance, and polymorphism

### Competency 7: Input and Output
The course includes:
- reading from files
- writing to files
- formatted output
- simple interactive programs using input and output

## Course Structure

The repository is organized by topic:
- 01_Getting Started
- 02_Control Flow
- 03_Data Collections
- 04_Memory & Text
- 05_Data Organization
- 06_Object-Oriented Programming
- 07_ Advanced C++

## How to Use This Repository

1. Start at the beginning and work through the lessons in order.
2. Read the comments and study notes before changing anything.
3. Try changing one value or one line at a time to see what happens.
4. Compile and run each file to reinforce the concept.

## Compiling a Lesson

From the repository root, you can compile a lesson like this:

```bash
g++ -std=c++11 "01_Getting Started (1-3)/01_HelloWorld.cpp" -o hello
```

Then run it with:

```bash
./hello
```

## Study Tips

- Read slowly and predict the output before running the program.
- Change one thing at a time so you can see the effect.
- Keep notes about mistakes and what caused them.
- Practice writing small variations of each example.

## Notes for Students

These files are meant to be a study guide as much as a code reference. The goal is not only to show syntax, but also to help students understand why each idea matters.
